Endpoints
GET/audioextract info + CDN url · 5h cache per video ID
GET /audio?q=dQw4w9WgXcQ → { id, title, uploader, duration, thumbnail, audio_url, proxy_url, host_url, cached, cache_expires }
Cache: resolved links stored 5h in data.json keyed by video ID. Cache hit returns instantly with cached: true and expiry timestamp.
GET/proxystream relay — range-aware, no disk
GET /proxy?q=dQw4w9WgXcQ → streams audio/webm (uses cached CDN url if available)
GET/hostdownload to disk → permanent URL
GET /host?q=ID → 202 { status:"processing", job_id, status_url } GET /job-status?id=X → { status, elapsed, file_url } GET /file?id=X → audio/webm download
GET/api/cacheview / manage 5h cache
GET /api/cache → { count, ttl_hours:5, entries[] } DELETE /api/cache → clear all cache DELETE /api/cache/:id → evict single entry
GET/stats & /exportlive metrics · download data.json
GET /stats → { uptime, total_requests, success_rate, cache_entries, hourly[24], logs[] } GET /export → full data.json download
